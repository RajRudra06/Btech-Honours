Algorithm Analysis

Breadth-First Search (BFS)

Strengths:

  • Guarantees shortest path in unweighted graphs
  • Complete - always finds solution if exists
  • Systematic level-by-level exploration

Weaknesses:

  • High memory usage - stores all nodes at current depth
  • Explores many unnecessary nodes
  • No heuristic guidance

Best Used: When shortest path is critical and graph is small/medium sized.

Depth-First Search (DFS)

Strengths:

  • Low memory - stores only current path
  • Fast for finding any solution
  • Efficient in maze-like environments

Weaknesses:

  • No shortest path guarantee
  • Can explore long wrong paths
  • May find suboptimal solutions

Best Used: When any solution acceptable and memory limited.

A* Search

Strengths:

  • Optimal with admissible heuristic
  • Fewer nodes explored than BFS
  • Best speed/optimality balance
  • Highly practical

Weaknesses:

  • Requires good heuristic
  • More memory than DFS
  • More complex implementation

Best Used: When shortest path needed with good heuristic (most real applications).
