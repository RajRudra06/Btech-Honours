DETAILED ANALYSIS OF ALL 6 PS

PS_1: Architecture-Driven Trade-offs in Semantic Segmentation: Boundary Precision vs Region Accuracy under Real-World Degradations (on point cloud ?)
SECTION 1: Research Framing
Field: Semantic Segmentation (2D Dense Prediction)
Main Lens: Architecture comparison
Supporting Lenses: Robustness analysis + Data-centric error analysis
Degradations:
	•	Geometric: Gaussian blur (motion/defocus), resolution downscaling
	•	Noise: Gaussian noise (sensor noise), salt-and-pepper noise
	•	Combined: Blur + noise (realistic camera conditions)
Key Research Questions:
	•	Do skip connections preserve boundary precision at the cost of region consistency?
	•	Does multi-scale context aggregation maintain semantic coherence in regions but blur boundaries?
	•	How do these trade-offs manifest differently across degradation types?
	•	Which architectural choices create inherent robustness vs learned robustness?
Hypothesis:
	•	Skip connections = high spatial detail retention → better boundaries, but more sensitive to noise propagation → worse regions under noise
	•	Dilated convolutions = large receptive fields → better semantic consistency in regions, but lose fine spatial detail → worse boundaries under blur

SECTION 2: Technical Implementation Details
MODELS (Architectural Families)
Model 1: U-Net with ResNet-50 Encoder
	•	Architecture philosophy: Strong encoder-decoder with symmetric skip connections
	•	Key components: 
	•	ResNet-50 encoder (pretrained on ImageNet)
	•	4 skip connections (conv1, layer1, layer2, layer3 → decoder blocks)
	•	Decoder: Upsampling (bilinear/transposed conv) + concatenation with skip features
	•	Final layer: 1x1 conv to num_classes
	•	Inductive bias: Spatial detail preservation, local feature propagation
	•	What you can vary: 
	•	Skip connection strength (multiply skip features by learnable weight)
	•	Number of skip connections (ablation: use only 2 skips vs all 4)
	•	Decoder depth (shallow vs deep decoder)
	•	Upsampling method (bilinear vs transposed conv)
Model 2: DeepLabV3+ with ResNet-50 Backbone
	•	Architecture philosophy: Context-heavy, multi-scale feature aggregation
	•	Key components: 
	•	ResNet-50 backbone (modified with atrous/dilated convolutions)
	•	ASPP (Atrous Spatial Pyramid Pooling): parallel dilated convs at rates [6, 12, 18] + global pooling
	•	Decoder: 1x upsampling + low-level feature fusion + 4x upsampling
	•	Output stride: 16 (controls how much spatial resolution is lost)
	•	Inductive bias: Semantic consistency, multi-scale context
	•	What you can vary: 
	•	ASPP dilation rates (e.g., [6,12,18] vs [3,6,9] for smaller receptive fields)
	•	Output stride (8 vs 16 — controls resolution vs speed trade-off)
	•	Decoder complexity (simple vs with skip from low-level features)
	•	Backbone depth (ResNet-50 vs ResNet-101)
Model 3: Lightweight — BiSeNet-V2 or DDRNet-23-Slim
	•	Architecture philosophy: Efficiency-biased, real-time inference
	•	Key components (BiSeNet-V2): 
	•	Dual-path: Detail Branch (high-res, shallow) + Semantic Branch (low-res, deep)
	•	Bilateral Aggregation Layer: Fuses detail and semantic features
	•	Lightweight backbone: Custom efficient blocks (not ResNet)
	•	Key components (DDRNet): 
	•	Dual-resolution network: High-res and low-res streams in parallel
	•	Bilateral fusion at multiple stages
	•	Deep aggregation pyramid pooling
	•	Inductive bias: Speed-accuracy trade-off, less redundancy
	•	What you can vary: 
	•	Branch depth (Detail vs Semantic branch capacity)
	•	Fusion method (how detail and semantic features combine)
	•	Backbone choice (MobileNetV2 vs ShuffleNetV2 vs custom)
DATASETS
Primary Choice: Cityscapes
	•	Why: Urban driving scenes, 19 classes, high-quality labels, standard benchmark
	•	Splits: 2975 train, 500 val, 1525 test (use train+val for training, test for eval if labels available, else use val)
	•	Resolution: 2048x1024 (downsample to 1024x512 or 512x256 for efficiency)
	•	Classes: Road, sidewalk, building, wall, fence, pole, traffic light, person, car, etc.
	•	Degradation protocol: 
	•	Train: Clean data only
	•	Eval: Clean + 4 degradation levels per type (mild, moderate, strong, severe)
	•	Blur: Gaussian kernel σ = [1, 2, 4, 6]
	•	Noise: Gaussian std = [10, 20, 40, 60] (0-255 scale)
	•	Resolution: Downsample factors = [2x, 4x, 8x, 16x] then upsample back
Alternative: BDD100K
	•	Why: Diverse weather/lighting (day, night, rain, fog), 10 classes
	•	Size: 70K train, 10K val
	•	Benefit: Natural degradations (rain/fog) already in dataset — can test generalization to real degradations
	•	Use case: If you want to add "real degradation" comparison to synthetic degradations
Backup: KITTI (if autonomous driving focus)
	•	Smaller dataset: ~200 annotated images
	•	Use case: Fine-grained analysis on smaller, cleaner dataset
	•	Limitation: Too small for full training — use pretrained models only
ARCHITECTURAL COMPONENTS TO VARY (Ablation Studies)
1. Skip Connections (U-Net focus):
	•	Baseline: All 4 skip connections
	•	Ablation 1: No skips (pure encoder-decoder)
	•	Ablation 2: Only shallow skips (conv1, layer1)
	•	Ablation 3: Only deep skips (layer2, layer3)
	•	Discovery: Which encoder depth contributes most to boundary preservation?
2. Multi-Scale Context (DeepLab focus):
	•	Baseline: ASPP with rates [6, 12, 18]
	•	Ablation 1: Single scale (no ASPP, just 3x3 conv)
	•	Ablation 2: Reduced ASPP rates [3, 6, 9] (smaller receptive fields)
	•	Ablation 3: Aggressive ASPP [12, 24, 36] (larger receptive fields)
	•	Discovery: Does more context help or hurt under different degradations?
3. Decoder Depth:
	•	Shallow decoder: 1-2 upsampling stages
	•	Deep decoder: 4+ upsampling stages with residual connections
	•	Discovery: Does decoder complexity affect robustness?
4. Normalization Layers:
	•	Batch Norm (standard)
	•	Group Norm (less batch-dependent)
	•	Instance Norm (more invariant to style/lighting)
	•	Discovery: Does normalization choice affect photometric robustness? (relevant if combining PS_1+PS_2)
METRICS (What You Measure)
Standard Metrics:
	•	Mean IoU (mIoU): Overall segmentation quality
	•	Per-class IoU: Which classes degrade most (small objects vs large regions)?
	•	Pixel Accuracy: Simple correctness measure
Boundary-Specific Metrics (YOUR NOVELTY): 4. Boundary F1 Score:
	•	Compute boundaries from ground truth masks (edge detection)
	•	Compute boundaries from predicted masks
	•	Measure precision/recall within distance threshold (e.g., 2-5 pixels)
	•	F1 = 2 * (precision * recall) / (precision + recall)
	•	Trimap IoU: 
	•	Divide image into 3 regions: interior (far from boundaries), boundary (near edges), exterior
	•	Compute IoU separately for boundary region
	•	Shows if model is failing at edges specifically
Region-Specific Metrics: 6. Interior IoU:
	•	Mask out boundary regions (e.g., 10-pixel band around edges)
	•	Compute IoU only on interior pixels
	•	Shows semantic consistency within objects
Degradation Robustness Metrics: 7. Relative mIoU Drop:
	•	(mIoU_clean - mIoU_degraded) / mIoU_clean
	•	Measures fragility
	•	Degradation Resilience Score: 
	•	Area under curve when plotting mIoU vs degradation severity
	•	Higher = more robust across all degradation levels
WHAT YOU COMPARE BETWEEN MODELS
Quantitative Comparisons:
	•	Overall robustness: mIoU drop across all degradations
	•	Boundary preservation: Boundary F1 under blur (skip connections should win)
	•	Region consistency: Interior IoU under noise (multi-scale context should win)
	•	Small object handling: Per-class IoU for small classes (pole, traffic light) — which architecture preserves these?
	•	Class-specific failure modes: Which classes degrade most per architecture?
Qualitative Analysis (Visual Inspection):
	•	Boundary bleeding: Do edges become fuzzy? (U-Net should be sharper)
	•	Region fragmentation: Do large objects break into pieces? (DeepLab should avoid this)
	•	Small object disappearance: Do traffic lights vanish under blur?
	•	Noise propagation: Do noisy artifacts appear in flat regions (roads)? (Skip connections propagate this)
Ablation Insights:
	•	Skip connection ablation: Plot boundary F1 vs number of skips
	•	ASPP ablation: Plot region IoU vs receptive field size
	•	Efficiency analysis: Plot mIoU vs FLOPs for lightweight models

PS_2: Photometric Robustness in CNN-Based Semantic Segmentation: How Architectural Choices Handle Lighting and Colour Distribution Shifts
SECTION 1: Research Framing
Field: Semantic Segmentation
Main Lens: Architecture comparison
Supporting Lens: Robustness to photometric degradations
Degradations:
	•	Lighting intensity: Overexposure (gamma > 1), underexposure (gamma < 1)
	•	Day/Night: Extreme brightness reduction + color temperature shift
	•	Shadows: Local darkening (simulate cast shadows)
	•	Color shifts: Temperature (warm/cool), tint (green/magenta cast)
Key Research Questions:
	•	Which architectures learn lighting-invariant features vs memorizing pixel intensities?
	•	Do skip connections propagate photometric bias from early (color-sensitive) layers?
	•	Does deep encoder abstraction naturally create lighting invariance?
	•	How do normalization layers (BN, GN, IN) affect photometric robustness?
Hypothesis:
	•	Shallow skip connections propagate raw RGB values → sensitive to lighting
	•	Deep encoders learn semantic features (edges, textures) → more lighting-invariant
	•	Instance Normalization should help more than Batch Norm for lighting shifts

SECTION 2: Technical Implementation
MODELS (Same as PS_1 but with normalization variants)
Model 1: U-Net + ResNet-50 (Baseline with Batch Norm)
Model 1b: U-Net + ResNet-50 (Instance Norm variant)
	•	Replace all Batch Norm layers with Instance Norm
	•	Why: Instance Norm normalizes per-sample, per-channel → more style-invariant
	•	Hypothesis: Should be more robust to lighting changes
Model 2: DeepLabV3+ (Batch Norm baseline)
Model 2b: DeepLabV3+ (Group Norm variant)
	•	Replace Batch Norm with Group Norm (num_groups=32)
	•	Why: Group Norm is batch-size independent, might generalize better
Model 3: Lightweight (BiSeNet-V2)
DATASETS
Primary: BDD100K
	•	Why: Has natural day/night splits, diverse lighting conditions
	•	Classes: 10 classes (road, car, person, traffic sign, etc.)
	•	Natural degradations: Already contains day/night/dawn/dusk images
	•	Degradation protocol: 
	•	Train: Daytime images only (clean, well-lit)
	•	Test: Day (clean), Night (natural), plus synthetic degradations on day images
Secondary: Cityscapes
	•	Use for synthetic degradations (no natural night images)
	•	Apply photometric transforms to create controlled experiments
Tertiary: Dark Zurich
	•	Nighttime dataset: Paired day-night images of Zurich
	•	Use case: Test extreme low-light generalization
	•	Size: Small (2,416 day, 2,920 night unlabeled, 201 night labeled)
	•	Limitation: Small labeled set, use for qualitative analysis only
PHOTOMETRIC DEGRADATIONS (How to Generate)
1. Global Illumination Changes:
# Gamma correction (controls overall brightness)
degraded = np.power(image / 255.0, gamma) * 255.0
# gamma > 1: darker (underexposure)
# gamma < 1: brighter (overexposure)
# Levels: gamma = [0.4, 0.6, 1.5, 2.0]
2. Color Temperature Shifts:
# Warm (orange/yellow cast) — simulate sunset
image[:,:,0] *= 1.2  # increase red
image[:,:,2] *= 0.8  # decrease blue

# Cool (blue cast) — simulate overcast
image[:,:,0] *= 0.8  # decrease red
image[:,:,2] *= 1.2  # increase blue
3. Simulated Shadows:
# Create local darkening masks
# Apply multiplicative factor < 1 to random regions
# Simulate cast shadows from buildings/trees
4. Day-to-Night Simulation:
# Severe underexposure + blue color shift
night_image = np.power(image / 255.0, 2.5) * 255.0  # darken
night_image[:,:,2] *= 1.3  # add blue tint
# Add noise (low-light sensor noise)
ARCHITECTURAL COMPONENTS TO VARY
1. Normalization Layer Type:
	•	Batch Norm: Standard, batch-dependent
	•	Instance Norm: Per-sample normalization, style-invariant
	•	Group Norm: Hybrid, batch-independent
	•	Layer Norm: Full channel normalization
	•	Experiment: Train 4 variants of U-Net (BN, IN, GN, LN) and compare photometric robustness
2. Skip Connection Depth (Same as PS_1):
	•	Hypothesis: Shallow skips (conv1) propagate RGB → lighting-sensitive
	•	Deep skips (layer3) propagate semantic features → more invariant
3. Color Space:
	•	Baseline: RGB input
	•	Variant: Convert to LAB or HSV before feeding to network
	•	Why: LAB separates luminance (L) from color (a, b) → might be more invariant
	•	Experiment: Train U-Net on LAB vs RGB, test lighting robustness
METRICS
Standard:

	•	mIoU on each photometric degradation
Lighting-Specific:
	•	Photometric Robustness Score:
	•	Average mIoU across all lighting conditions
	•	Higher = more lighting-invariant
	•	Class-wise Lighting Sensitivity:
	•	Which classes fail most under low-light? (pedestrians, signs)
	•	Which are robust? (road, sky)
	•	Boundary Stability Under Lighting:
	•	Does boundary F1 drop more under shadows than global darkening?
	•	Shadows create local contrast changes → edges might be more affected
Visualization:
	•	Activation maps: Visualize early vs late layer features under lighting changes
	•	Do early layers respond to brightness? Do late layers ignore it?
WHAT YOU COMPARE

PS_3: Geometric Perception Under Photometric Stress: Surface Normal Estimation Robustness to Lighting Variation Across CNN Architectures
SECTION 1: Research Framing
Field: Surface Normal Estimation (3D Low-Level Vision)
Main Lens: Architecture comparison
Supporting Lens: Robustness to photometric variation
Task: Given single RGB image → predict 3D surface orientation at each pixel (normal vector in camera coordinates)
Degradations:
	•	Shadows (local lighting changes)
	•	Overexposure (loss of texture detail)
	•	Underexposure (loss of geometric cues)
	•	Color temperature shifts (warm/cool lighting)
Key Research Questions:
	•	Which architectures disentangle 3D geometry from 2D lighting/appearance?
	•	Do skip connections help (preserve geometric detail) or hurt (propagate photometric noise)?
	•	How do boundary normals (edges, discontinuities) degrade vs planar normals (flat surfaces)?
	•	Can deep encoders learn lighting-invariant geometric features?
Hypothesis:
	•	Deep encoders (ResNet-101) learn shape-from-shading that's lighting-invariant
	•	Skip connections propagate pixel-level lighting info → hurt under shadows
	•	Planar surfaces (walls, floors) maintain normals under lighting changes
	•	Edges/discontinuities (corners, object boundaries) fail when shadows obscure depth cues

SECTION 2: Technical Implementation
MODELS
Model 1: U-Net with ResNet-50 Encoder
	•	Architecture: 
	•	ResNet-50 encoder (pretrained on ImageNet)
	•	4 skip connections
	•	Decoder with upsampling
	•	Output: 3-channel (Nx, Ny, Nz normal components)
	•	Output activation: L2 normalization (ensure unit vectors)
	•	Loss function: Cosine similarity loss or L1 loss on normalized vectors
Model 2: ResNet Encoder-Decoder (No Skips)
	•	Same encoder, but pure encoder-decoder (no skip connections)
	•	Hypothesis: Less sensitive to lighting because no raw pixel propagation
	•	Trade-off: Might lose fine geometric detail (edges)
Model 3: Lightweight — MobileNetV2 Encoder + Decoder
	•	Efficient architecture
	•	Hypothesis: Lower capacity → might fail at complex geometry under lighting stress
Model 4 (Optional): DPT (Dense Prediction Transformer)
	•	Transformer-based (Vision Transformer encoder)
	•	Hypothesis: Global context via self-attention → more lighting-invariant
	•	Trade-off: Heavier, slower, but might be more robust
DATASETS
Primary: NYUv2 (Indoor Scenes)
	•	Size: 1,449 RGB-D images (795 train, 654 test)
	•	Ground truth: Depth maps (can compute normals from depth)
	•	Scenes: Indoor rooms (furniture, walls, floors)
	•	Why: Standard benchmark, diverse geometry, controlled lighting
	•	Limitation: Limited lighting variation (mostly indoor lighting)
Secondary: ScanNet
	•	Size: Larger (1,513 scenes, ~2.5M frames)
	•	Ground truth: 3D reconstructions → compute normals
	•	Benefit: More data, diverse indoor environments
	•	Limitation: Requires preprocessing to extract normals
Tertiary: DIODE (Dense Indoor/Outdoor DEpth)
	•	Size: 8,574 indoor, 16,884 outdoor images
	•	Ground truth: Laser-scanned depth
	•	Benefit: Indoor + outdoor, more diverse lighting
	•	Use case: Test generalization to outdoor lighting
Degradation Protocol:
	•	Train: Clean images from NYUv2/ScanNet
	•	Test: Apply photometric degradations (same as PS_2: gamma, shadows, color shifts)
	•	Real degradation test: If DIODE has outdoor images, test on those (natural sunlight variation)
NORMAL COMPUTATION FROM DEPTH (If Using Depth Datasets)
# Given depth map D

# Example:

normal = normalize([-dz_dx, -dz_dy, 1])

PS_4: Attention Under Adversity: How Salient Object Detection Architectures Respond to Visual Clutter and Image Quality Degradation
SECTION 1: Research Framing
Field: Salient Object Detection
Main Lens: Architecture comparison (attention mechanisms)
Supporting Lenses: Robustness to scene complexity + image quality degradation
Task: Given RGB image → binary/continuous saliency map (highlight most attention-grabbing object)
Two Experimental Variables:
	•	Scene complexity: Simple (single object, plain background) vs Cluttered (multiple objects, textured background)
	•	Image quality: Clean vs Degraded (blur, noise, low resolution)
Key Research Questions:
	•	Does explicit attention help or hurt in cluttered scenes?
	•	Do attention mechanisms focus on wrong objects in clutter (false positives)?
	•	Does multi-scale fusion provide robustness by averaging out distractions?
	•	How does degradation interact with clutter? (Double stress test)
Hypothesis:
	•	Attention = precise in simple scenes, distracted in clutter
	•	Multi-scale fusion = robust in clutter (averages features), but less precise
	•	Degradation + clutter = catastrophic for attention (confusion amplified)

SECTION 2: Technical Implementation
MODELS
Model 1: BASNet (Boundary-Aware Salient Object Detection)
	•	Architecture: 
	•	Encoder: ResNet-like with residual blocks
	•	Decoder: Refinement module with residual connections
	•	Attention: Hybrid attention module (channel + spatial attention)
	•	Multi-scale supervision (deep supervision at multiple decoder stages)
	•	Inductive bias: Explicit attention for focus, boundary refinement
Model 2: U²-Net (U-Net with U-blocks)
	•	Architecture: 
	•	Nested U-Net structure (U-Net inside U-Net)
	•	Each encoder/decoder block is itself a mini U-Net
	•	Attention: Implicit (via nested architecture, captures multi-scale)
	•	Inductive bias: Deep multi-scale feature extraction without explicit attention
Model 3: PoolNet (Pooling-Based Multi-Scale Fusion)
	•	Architecture: 
	•	ResNet-50 backbone
	•	Global Guidance Module (GGM): Pooling-based global context
	•	Feature Aggregation Module (FAM): Multi-scale fusion via pooling
	•	No explicit attention: Uses pooling for implicit context
	•	Inductive bias: Multi-scale averaging, robust but less precise
Model 4 (Lightweight): MobileNet-based Saliency
	•	Custom lightweight architecture
	•	Inductive bias: Efficiency over accuracy
DATASETS
Primary: DUTS (Densely Annotated UTah Saliency)
	•	Size: 10,553 train, 5,019 test
	•	Ground truth: Pixel-level saliency masks
	•	Scene diversity: Indoor/outdoor, simple/complex
	•	Why: Large, standard benchmark, diverse complexity
Secondary: DUT-OMRON
	•	Size: 5,168 images
	•	Characteristics: Specifically contains cluttered scenes with multiple salient objects
	•	Use case: Test clutter robustness
Tertiary: ECSSD (Extended Complex Scene Saliency Dataset)
	•	Size: 1,000 images
	•	Characteristics: Complex scenes, multiple objects
	•	Use case: Another clutter test set
Scene Complexity Protocol:
	•	Simple: Filter images with <3 objects, clean backgrounds (manual or automatic filtering)
	•	Cluttered: Images with 5+ objects, textured backgrounds
	•	OR: Use DUT-OMRON (naturally cluttered) vs ECSSD simple subset
Degradation Protocol:
	•	Same as PS_1: Blur, noise, resolution loss
	•	Apply to both simple and cluttered scenes
	•	Creates 2x4 = 8 test conditions (2 complexity × 4 degradations)
ARCHITECTURAL COMPONENTS TO VARY
1. Attention Mechanism Type:
	•	Channel attention: Squeeze-and-Excitation (SE) blocks
	•	Spatial attention: Convolutional attention maps
	•	Hybrid: Channel + spatial (CBAM-style)
	•	Ablation: Remove attention modules from BASNet, see impact
2. Multi-Scale Fusion Strategy:
	•	Simple concatenation: Stack features from different scales
	•	Weighted fusion: Learn fusion weights
	•	Pyramid pooling: Use pooling (PoolNet style)
	•	Ablation: Single-scale vs multi-scale prediction
3. Decoder Complexity:
	•	Shallow decoder (1-2 stages)
	•	Deep decoder (4+ stages with refinement)
	•	Discovery: Does refinement help in clutter?
4. Global Context Module:
	•	Global average pooling
	•	Non-local attention (self-attention across spatial locations)
	•	Pyramid pooling (multi-scale global context)
	•	Discovery: Does global context reduce clutter distraction?
METRICS
Standard Saliency Metrics:
	•	F-measure (Max F-score): 
	•	Precision-recall trade-off
	•	Higher = better boundary precision and region completeness
	•	MAE (Mean Absolute Error): 
	•	Pixel-wise error between predicted and ground truth
	•	Lower = better
	•	S-measure (Structure Similarity): 
	•	Considers structural similarity (object-level and region-level)
	•	E-measure (Enhanced Alignment): 
	•	Combines local and global alignment
Clutter-Specific Metrics (YOUR NOVELTY): 5. False Positive Rate in Clutter:
	•	How many background objects are incorrectly highlighted?
	•	Compute FP specifically in cluttered regions
	•	Object Separation Quality: 
	•	Can the model isolate the most salient object from distractors?
	•	Measure IoU of most salient object only
	•	Attention Consistency: 
	•	For attention-based models: Visualize attention maps
	•	Measure attention entropy (low = focused, high = scattered)
	•	Discovery: Attention becomes diffuse in clutter
Degradation-Clutter Interaction: 8. Clutter Amplification Score:
	•	(Error_cluttered_degraded - Error_simple_degraded) - (Error_cluttered_clean - Error_simple_clean)
	•	Measures if degradation hurts more in clutter
	•	Discovery: Attention models show higher amplification
WHAT YOU COMPARE
Quantitative:
	•	Attention vs Multi-scale: BASNet (attention) vs PoolNet (pooling) in clutter
	•	Simple vs Cluttered: F-score drop in cluttered scenes per architecture
	•	Degradation impact: Which models are most robust to blur/noise?
	•	Interaction effect: Does clutter + degradation double the error, or more?
Qualitative (Visual Analysis):
	•	Attention maps: Where does BASNet focus in clutter? (Visualize attention heatmaps)
	•	Failure modes: 
	•	False positives: Background objects incorrectly highlighted
	•	False negatives: Salient object missed in clutter
	•	Boundary errors: Object edges blurred in degraded images
	•	Saliency map evolution: How does predicted saliency change from simple → cluttered → degraded?
Ablation Studies:
	•	Remove attention from BASNet → measure precision loss
	•	Single-scale vs multi-scale in PoolNet → measure robustness

PS_5: Monocular Depth Estimation Under Adverse Conditions: Architecture Robustness to Weather and Low-Light Degradations
SECTION 1: Research Framing
Field: Monocular Depth Estimation (3D Vision)
Main Lens: Architecture comparison (CNN vs Transformer)
Supporting Lens: Robustness to weather and lighting
Task: Single RGB image → depth map (distance from camera to each pixel)
Degradations:
	•	Weather: Rain, fog, snow (occlusion-based)
	•	Lighting: Nighttime, low-light, extreme shadows
Key Research Questions:
	•	Do transformers' global attention make them more robust to local occlusions (rain/fog)?
	•	Do CNNs' local receptive fields fail when occluded patches are nearby?
	•	How do skip connections propagate noise/occlusion errors in depth estimation?
	•	Which architectures maintain geometric consistency (depth edges align with image edges) under degradation?
Hypothesis:
	•	Transformers robust to fog/rain (global context averages out occlusions)
	•	CNNs fail under local occlusions (limited receptive field)
	•	Skip connections hurt under low-light (propagate dark/noisy pixels)
	•	Depth edges (discontinuities) degrade more than smooth depth regions

SECTION 2: Technical Implementation
MODELS
Model 1: MiDaS (Mixed Data Sampling) — CNN-based
	•	Architecture: 
	•	Encoder: ResNet-101 or EfficientNet
	•	Decoder: Feature pyramid-style with skip connections
	•	Multi-scale features aggregated
	•	Inductive bias: Local features, strong skip connections
	•	Pretrained: Available on HuggingFace
Model 2: DPT (Dense Prediction Transformer)
	•	Architecture: 
	•	Encoder: Vision Transformer (ViT-Base or ViT-Large)
	•	Decoder: Reassemble + Fusion modules
	•	No skip connections (transformer encoder is not hierarchical like CNNs)
	•	Inductive bias: Global self-attention, long-range dependencies
	•	Pretrained: Available on HuggingFace
Model 3: DepthAnything (Recent SOTA)
	•	Architecture: 
	•	Encoder: Vision Transformer (DINOv2-based)
	•	Decoder: DPT-style
	•	Trained on massive diverse data
	•	Inductive bias: Strong generalization, global attention
	•	Pretrained: HuggingFace
Model 4 (Hybrid): BEiT-Adapter or similar
	•	Combines transformer encoder with CNN-style decoder
	•	Discovery: Does this combine benefits of both?

PS_6: Efficiency-Robustness Trade-offs in Real-Time Semantic Segmentation: How Model Compression Affects Degradation Resilience
SECTION 1: Research Framing
Field: Semantic Segmentation (Real-Time / Efficient Models)
Main Lens: Efficiency trade-offs
Supporting Lens: Robustness analysis + Architecture comparison
Two-Axis Analysis:
	•	X-axis: Efficiency (FLOPs, parameters, inference time, memory)
	•	Y-axis: Robustness (performance under degradations)
Key Research Questions:
	•	Do lightweight models collapse faster under degradations (less redundancy)?
	•	Is there a "sweet spot" for real-time deployment (good speed + robustness)?
	•	Which efficient design patterns (depthwise conv, attention, pruning) maintain robustness?
	•	Can we quantify the efficiency-robustness Pareto frontier?
Hypothesis:
	•	Heavy models are more robust (more parameters = more redundancy)
	•	Lightweight models are fragile (less capacity to handle distribution shift)
	•	BUT some efficient designs (e.g., efficient attention) might be surprisingly robust
	•	Trade-off curve: As FLOPs decrease, robustness drops, but non-linearly
